# Nginx

高性能http代理和反向代理服务器，内存少 性能优化 5w 并发

# 特性

  • 反向代理

    • 正向代理：客户端需要配置代理服务器，才可以访问；
    • 反向代理：客户端访问服务器，先到反向代理服务器，客户端无感知；

  • 负载均衡

    • 多个请求并发到服务器，需要增加服务器数量。Nginx通过配置，可以分发到各个服务器中；策略list
      • 轮询策略：每个请求按照时间顺序分配到不同的服务器，如果有服务器宕机，直接从list中删除；
      • weight：权重越高，分配概率越大
      • ip_hash：按照ip的hash结果，每个ip固定访问一个服务器地址；
      • fair：根据请求的响应时间来分配，优先选择响应早的；

  • 动静分离(分配)

    • 动态资源：服务器用来部署jsp、serverlet等
    • 静态资源：文件js、css

# 配置文件

  • 全局块

    • worker_processes

  • event

    • worker_connection 最大连接数

  • http模块

    • upstream
    • server
      • proxy_pass
upstream {
    // 负载均衡
    upstream serverName {
        server http1 weight=1;
        server http2 weight=1;
    }

    server {
        listen 9001
        // 配置静态资源
        location /static {
            root /data/static;
        }
        location / {
            proxy_pass http://serverName
        }
        // backupServer 1
        location ~ /a {
            proxy_pass http://xxx.xx.xx.xx:8080
        }
        // backupServer 2
        location ~ /b {
            proxy_pass http://xxx.xx.xx.xx:8081
        }
    }
}
