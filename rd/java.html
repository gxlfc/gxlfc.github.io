  • JDK（Java Development Kid，Java 开发开源工具包），是针对 Java 开发人员的产品，是整个 Java 的核心，包括了 Java 运行环境 JRE、Java 工具和 Java 基础类库。

  • JVM（Java Virtual Machine，Java 虚拟机）是整个 Java 实现跨平台（windows、mac、linux）的最核心的部分，能够运行以 Java 语言写作的软件程序。

# 数据类型

  • 基本类型（字节大小）
    • 数值
      • 整数：byte（8）、short（16）、int（32）、long（64）
      • 浮点：float（4）、double（8）
    • 字符：char（2）
    • 布尔：boolean（1）
  • 引用类型：类(class)、接口(interface)、数组([])

转换: 数值型数据的转换：byte→short→int→long→float→double 字符型转换为整型：char→int。

# 内存分配：

int[] arr = new int[3]; 栈内存：int[] arr + 存储局部变量 + 使用完毕，立刻消失 堆内存：new int[3] + 存储所有new出来的内容（实体、对象） + 每一个new出来的东西都有地址值，使用完毕，会在垃圾回收器空闲时被回收；

# 传参：

  • 基本类型的参数: 形式参数的改变，不影响实际参数的值;
int number = 1;
getNum(number);
System.out.print(number); // 1
public static void getNum(int number) {
    number = 2;
}
  • 引用类型的参数: 形式参数的改变，影响实际参数的值;
int[] arr = {1, 2, 3};
change(arr);
System.out.print(arr); // 100, 2, 3
public static void change(int[] arr) {
    arr[0] = 100;
}

# 原型链

java：所有类的基类Objecct javaScript： 原型链的终点Object.property

  • 值引用，类型引用

    • 任何对象变量的值都是针对对象内存地址的引用；所有对象都存储在堆中；

  • 实例域 实例方法

    • private String name;
    • private final String name; // final 实例域

  • 静态域 静态常量 静态方法

    • public static int test = 1; // 静态域属于这个类，而不属于这个类的某个对象，每个类中只有一个这样的域；
    • public static final int test = 1; // static final (缺一不可)则这个域就是一个静态常量；
    • 静态方法： 没有隐式参数，不能操作对象，只可以访问静态域；

# 抽象类

  • 不能实例化
  • 可以有构造函数，用来实现子类实例的初始化
  • 没有方法体的方法（() {}）定义为抽象类；public abstract int func();
  • 一个类中有抽象方法，该类需定义为抽象类，抽象类可以没有抽象方法；
  • 抽象类的子类
    • 继承所有的抽象方法
    • 子类本身就是一个抽象类 对类进行抽象，包括属性行为

# 接口类

  • 不能实例化
  • 接口的实现类
    • 重写接口中的所有抽象方法
    • 要么是抽象类
  • 内部的变量，默认被 public static final 修饰；
  • 成员方法：默认修饰符：public abstract；
  • 没有构造方法； 对行为进行抽象，主要是行为

抽象类是对事物的抽象，接口是行为的抽象； 一个类没有父类，继承自Object类；

# Iterable 对象

一个root interface，实现此接口对象就能使用 “for-each循环” 语句。

  • Iterator<T>iterator(), 返回一个内部元素为T类型的iterator
  • default void forEach(Consumer<? super T> action)，对Iterable的每个元素执行给定的操作，直到所有元素都被处理或动作引发异常。 除非实现类另有规定，否则按照迭代的顺序执行操作（如果指定了迭代顺序）。

# Iterator 对象

// 判断一个对象集合是否还有下一个元素
boolean hasNext();
// 获取下一个元素
E next();
// 删除最后一个元素。默认是不支持的，因为在很多情况下其结果不可预测，比如数据集合在此时被修改
default void remove(){...}
// 主要将每个元素作为参数发给action来执行特定操作
default void forEachRemaining(Consumer<? super E> action){...}

# demo：自己实现Iterator方法的Iterable对象。

public class Demo<E> implements Iterable<E> {
    @Override
    public Iterator iterator() {
        return new MyIterator();
    }

    private class MyIterator implements Iterator<E> {
        @Override
        public boolean hasNext() {
            return false;
        }
        @Override
        public E next() {
            return null;
        }
    }
}

Tips:

@Override是伪代码,表示重写。(不写也可以)，使用益处: 1、当做注释使用,方便阅读； 2、编译器可以验证@Override的方法名是否为父类所有，否则报错。
