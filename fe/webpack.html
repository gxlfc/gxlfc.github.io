# css-Loader

建议和 css-module共同食用。

# 插件： extract-text-webpack-plugin

# 三大疑问之二：what && why?

It moves all the required *.css modules in entry chunks into a separate CSS file. So your styles are no longer inlined into the JS bundle, but in a separate CSS file (styles.css). If your total stylesheet volume is big, it will be faster because the CSS bundle is loaded in parallel to the JS bundle.

给我翻译翻译（姜文电影桥段）：该插件可以将入口chunk的所有 *.css 放到一个单独的css文件中，这样你的css就不用和js杂放在一起。由于css和js并行加载，如果输出的整css文件内存很大，会加快页面渲染。

# 使用场景webpack plugins：options =>
new ExtractTextPlugin(options: filename | object)

object = {
    id, // id
    filename, // Name of the result file. 可以包含 [name], [id] and [contenthash],
    // 我们知道webpack最烦的一点就是文档很乱 & 插件之间的配置比较混乱，容易相互影响，总之
    // 默认情况下，它仅从初始chunk中提取, 改项为true时，也会从所有其他chunk中提取；
    // when use CommonsChunkPlugin，且ExtractTextPlugin.extract里包含人家要抽出来的公共 chunk 时,allChunks必须=true。
    allChunks: Boolean,
    disable: Boolean, // Disables the plugin
    ignoreOrder: Boolean, // 禁用顺序检查 (这对 CSS 模块很有用！)，默认是false
}
