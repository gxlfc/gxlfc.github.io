# CSS Module

我还记得一开始接触前端的时候，头最大的问题就是css命名，感觉翻牌最多的就是xx-container，xx-wrapper。随着前端JS各种技术突起，css感觉还在解放前。那天在开发一个vue项目的时候，偶尔检索了下css module，之前没接触过，抽空来补补课。网上资源千千万，还是原文文档更好看。

# 译文[CSS Modules]

我突然意识到一个问题，中英文夹杂的译文，我反而看的比较顺眼，也更易于理解。[手动摊手]

CSS的规则都是全局的，任何一个组件的样式规则，都对整个页面有效。产生局部作用域的唯一方法，就是使用一个独一无二的class的名字，不会与其他选择器重名，这就是 CSS Modules 产生的原因。

  • CSS模块是一个CSS文件，默认情况下，所有class names 和 animation names 都在本地作用域内。
  • CSS中涉及的所有URL（url（...））和@import均采用模块请求格式（./xxx和../xxx表示相对，xxx和xxx / yyy表示在模块文件夹中，即在node_modules中）。最佳实践：vue-loader

css modules 可以编译为 ICSS or Interoperable CSS（一种低阶交换格式？），但其书写方式和普通css类似；

/* style.css */
.className {
  color: green;
}

当从js模块引入css模块时，导出的是一个对象，并且该对象具有从local name 和 global name的所有映射。

import styles from "./style.css";
// import { className } from "./style.css";
element.innerHTML = '<div class="' + styles.className + '">';

# Naming

对于本地className，建议使用驼峰命名，但不强制使用。

推荐这样做是因为，当尝试以.访问style.class-name时，kebab-casing可能会导致意外的行为。可以用带有括号符号的kebab-case（例如style ['class-name']）解决问题，但style.className更干净。

# Exceptions

:global 可以让你从当前的选择器标识切换到全局作用域， global（.xxx） respective @keyframes：global（xxx）表明了xxx是在全局范围内。同理， :local and :local(...) for 局部作用域.

If the selector is switched into global mode, global mode is also activated for the rules. (This allows us to make animation: abc; local.) 比如: .localA :global .global-b .global-c :local(.localD.localE) .global-d

# Composition

.className {
  color: green;
  background: red;
}

.otherClassName {
  composes: className;
  color: yellow;
}

可以有多个组合，但组合必须放在其他样式之前。扩展仅适用于局部作用域的选择器，并且仅在选择器为单个类名称时才适用。当一个类名组成另一个类名时，CSS模块将导出本地类的两个类名。可使用composs组合多个类：classNameA classNameB;。

# Dependencies

# Composing from other files

可以从其他css模块中引入类，构成组合类；

.otherClassName {
  composes: className from "./style.css";
}

注意，如果从多个文件中引入css模块，其之间的依赖关系是不确定的，请确保不要为来自不同文件的多个类名中的同一属性定义不同的值。 注意，不要造成循环依赖，否则属性之间的覆盖性是不确定的。模块系统也有可能触发报错；

最好的做法是：类仅做一件事情并且依赖项是分层的。

# Composing from global class names

.otherClassName {
  composes: globalClassName from global;
}

# Usage with preprocessors

Preprocessors can make it easy to define a block global or local.i. e. with less.js

:global {
  .global-class-name {
    color: green;
  }
}

# 实际应用

如上讲述了css module的概念，如下结合实际项目做功能描述，css编译的时候，通常为了保证其属性的独立性/局部性，会做一次编译，比如增加后缀hash。

# 1、webpack结合使用：css-loader

webpack中的css-loader，其中的modules参数，表示是否开启 CSS 模块；如下：

















 









module.exports = {
    module: {
        rules: [
            {
                // For pure CSS (without CSS modules)
                test: /\.css$/i,
                use: ['style-loader', 'css-loader'],
            },
            {
                // For CSS modules
                test: /\.module\.css$/i,
                use: [
                    'style-loader',
                    {
                        loader: 'css-loader',
                        options: {
                            modules: true,
                        }
                    }
                ]
            }
        }
    }
}

经过loader 编译之后，会用唯一的标识符(identifier)（带哈希值）来替换局部选择器，所选择的唯一标识符以模块形式被导出。

:local(.className) { background: red; }
:local .className { color: green; }
:local(.className .subClass) { color: green; }
:local .className .subClass :global(.global-class-name) { color: blue; }

._23_aKvs-b8bW2Vg3fwHozO { background: red; }
._23_aKvs-b8bW2Vg3fwHozO { color: green; }
._23_aKvs-b8bW2Vg3fwHozO ._13LGdX8RMStbBE9w-t0gZ1 { color: green; }
._23_aKvs-b8bW2Vg3fwHozO ._13LGdX8RMStbBE9w-t0gZ1 .global-class-name { color: blue; }

exports.locals = {
  className: '_23_aKvs-b8bW2Vg3fwHozO',
  subClass: '_13LGdX8RMStbBE9w-t0gZ1'
}
