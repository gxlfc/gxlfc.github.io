# 全栈之艰

  • dev热加载 => pm2
  • 数据库DB/ 缓存 Redis => 百度 node-ral
  • 日志追踪 => logger
  • 现有接口耦合
  • qsb、线上监控
  • 生产环境部署
  • 内存泄露（OOM） 多核CPU超限 => 现有框架支持度不是很高

# 中间层

参考资料：

  • https://2014.jsconfchina.com/slides/herman-taobaoweb/index.html#/69
  • http://fex.baidu.com/yog2/

前端与后端之间搭建nodejs中间层方案，以实现前后端职责上的完全分离，更清晰的，更加专注于各自的职责。阿里 - 玉伯。

  • 前端：负责View和Controller层。
  • 后端：只负责Model层，业务处理/数据等。
