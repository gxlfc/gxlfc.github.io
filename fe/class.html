# es6 - String

  • includes()：是否包含了参数字符串，返回布尔值
  • startsWith()：参数字符串是否在原字符串的头部，返回布尔值
  • endsWith()：参数字符串是否在原字符串的尾部，返回布尔值

# es6 - Array

  • 扩展运算符: ...[1, 2, 3] => 1, 2, 3
    • 数组最大值：Math.max(value1[,value2, ...]) => Math.max.apply(null, arr) => Math.max(...arr)
    • 合并数组：Array.prototype.push.apply(arr1, arr2) => arr1.push(...arr2) => [arr1, ...arr2]
  • Array.from(obj, mapFunc, thisArg) :类数组arrLike/可iterable对象转化为数组
    • es5： [].slice.call(arrLike)
    • 第二个参数是function，类似于map操作
  • Array.of(value1[,value2, ...]) => [value1, vulue2]
  • Array.find() 找到符合条件的第一个成员/undefined
  • Array.findIndex() 找到符合条件的第一个成员为止 / -1
  • Array.fill(value, startPos, endPos) 初始化数组比较有用
  • Array.entries()键值对、Array.keys()键名、Array.values()键值
  • Array.includes 数组是否含给定值，针对简单数组有效
  • 空位预警，数组中如果存在空位，视其为undefined；

# Class

你是否为原型链继承搞得头晕脑胀？你是否看JS高程看了几编还是云里雾里？从开始写java后，前端间隔1年后开始接触，发现其语言借助了太多其他语言的语法，而且做了很多trick，终于理解了为啥有人从其他语言转js写起来别扭了。

const testSymbol = Symbol('private orchid');
class Person {
    constructor(name) {
        this.name = name;
    }
    [testSymbol]() {

    }
}

作用域问题参见java-访问权限

  • 私有变量（private）：只有class内部可以使用，外部无法访问；
    • #来申明私有变量，依赖babel 7 和 stage 3 prev
  • 私有方法（private）
    • Symbol: 上述代码中的testSymbol方法，你想调用的时候，应该咋办？
    let orchid = new Person(); 
orchid[testSymbol];
    那么问题来了， testSymbol是啥？你得重新定义一个symbol。基于symbol的定义：a symbol value is created by invoking the function Symbol, which dynamically produces an anonymous, unique value。Symbol can have an optional description, but for debugging purposes only。此symbol非彼symbol，你在调用的时候使用的symbol已经非class定义的了。从而达到了私有防范的目的；
    • 外置到class 同级 待补充
  • 静态方法（static）：方法不会被实例继承，直接通过类调用，子类可以使用，实例不可使用；
  • 静态属性（static）：class本身的属性
class Person {
    static staticPro = 1；
    constructor(name) {}
}
  • 实例属性：实例可以读取到该属性；
class Person {
    newObjPro = 1；
    constructor(name) {}
}

# Module

模块规范大概有三种：

  • commonJs：服务器，没有带宽限制，同步加载，

    • require：获取依赖关系
    • exports：导出模块内容和模块标识符；
    • 运行时加载，才能确定模块依赖；
    // test.js
exports.doSth = function() { console.log(111); }
// 使用require，多个require串行引入；
var test = require('test.js');
exports.doOth = function() { 
    var b = test.doSth(); 
    return b + 'ee';
};

  • AMD：浏览器端，带宽限制，异步加载

    • 使用define函数d定义模块，允许你在函数定义之前声明此模块的相关依赖；
    • 运行时加载，才能确定模块依赖；
    define（'module / id / string'，['module'，'dependency'，'array']，function（module，dependency，array）
    // do sth
	return ModuleContents;
}）;
// 使用
require(['module / id / string'],function(module){
    // do sth
});

  • ES6 module：两者皆可；

    • export 导出模块
      • 输出的接口和其对应的值是动态绑定，通过该接口可以取到模块实时的值，也可以变更其值。
      // exprot.js
export var name = 'Edwin';
// main.js
import {test} from 'exprot.js'
setTimeout(() => {test = 'Orchid'}, 1000);
      • 如果模块整体加载的时候（import * as family from 'test.js'），不可以在运行做改变；
      • export default 命令（输出一个名叫default的变量）
        • 后面不能跟变量声明语句；
        • 适合于不想对导出模块命名的case，在import 可以对其命名。
        • 由于一个模块只能default使用一次，，所以import时，就不需要{}了，因为只有一个方法；
    • import 会变量提升, 先于模块内的内容执行
      • require 和 import 最好不要混合使用。
        • import在静态解析阶段执行
        • require在运行时执行
        • 如果再有模块依赖/异步同步逻辑的时候，会导致未可知结果的运行代码；
      • 动态引入：执行时才加载模块
      import('./module.js')
    .then((module) => {
        // do sth
    });
// or
let module = await import('./module.js');

  • 模块加载

<script src="" async> // 异步加载，加载完成时，开始执行，阻止页面渲染
<script src="" defer> // 异步加载，页面全部渲染完成，才开始执行
//es6 module
<script type="module" src="">  =  <script src="" defer>
