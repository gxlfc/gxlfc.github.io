# iterable：[Symbol.iterator]

  • 数据结构只要部署了iterator接口，默认的iterator接口部署在数据结构的Symbol.iterator属性 => 数据结构具有Symbol.iterator属性，则认为是iterable。
  • Symbol.iterator是返回Symbol的iterator属性，得到当前数据结构默认的遍历器生成函数。
  • 原生具备iterator接口的数据结构：Array、Map、Set、String、TypedArray、函数arguments对象、nodeList对象
    • 类数组（NodeList为例）部署方式：NodeList.prototype[Symbol.iterator] = Array.prototype[Symbol.iterator];NodeList本来就是iterable，代码执行后无副作用；
    • 其他对象想要用for of循环，需要自己在 [Symbol.iterator]属性实现；
    • 如果调用Symbol.iterator方法，得到的不是一个iterator对象，解释引擎会报错；


 

 







 


// iterable
const iterable = {
    [Symbol.iterator]: function() {
        return {
            next: function() {
                return {value: '', done: true}
            }
        }
    }
};
// 自带iterator接口
const a = [1,2,3];
const iterator = a[Symbol.iterator]();
iterator.next() => {valule: 1, done: false};

# 不得不说的for of 循环

在可迭代对象（ 1. Symbol.iterator的属性;2. 该属性是一个遍历器生成函数function， 内部实现了next方法; 原生自带该属性的包括 Array，Map，Set，String，TypedArray，arguments 对象、Generator、自定义迭代器生成函数的对象）上创建一个迭代循环，调用自定义迭代钩子，并为每个不同属性的值执行语句;

var iterable = {
    [Symbol.iterator]() {
        return {i: 0,
            next() {
                if (this.i < 3) {
                    return { value: this.i++, done: false };
                }
                return { value: undefined, done: true };
            }
        };
    }
};
for (var value of iterable) {
  console.log(value); // 1,2,3
}
  • 遍历顺序：按照各个成员被添加到数据结构的顺序；
    • set数据结构做遍历，item返回的是一个值；
    • map数据结构做遍历，item返回的是map成员的键名和键值；
let set = new Set('a', 'b');
let map = new Map().set('a', 1).set('b', 1);
for (var i of set) { console.log(i) } //a,b
for (var i of map) { console.log(i) } //['a', 1],['b', 2]
  • 数组遍历优缺点对比
for forEach for in for of
麻烦 跳不出循环，break/return无效 不仅可以遍历数字key，还可以遍历手动添加的key，为对象遍历而生。 无缺点
循环读取键名 读取键值

# 承上启下：generator结合

我们知道了iterator的实现，就是带有Symbol.iterator属性是生成器函数，内部实现了迭代逻辑 => next()方法逻辑，由于generator自带next逻辑，所以他本身就是一个生成器函数

var demo1 = {};
demo1[Symbol.iterator] = function* () {
    yield '1';
    yield '2';
}
